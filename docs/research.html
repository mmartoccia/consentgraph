Agents of Chaos: What the Red Team Found

March 11, 2026

20 researchers spent two weeks breaking autonomous AI agents. 11 exploits succeeded. Here's which ones are solvable today, which need ConsentGraph, and which are open problems nobody has answers for yet.

A new red-team study deployed LLM-powered agents with persistent memory, email, shell access, and multi-channel communication. Researchers probed them adversarially. The dominant attack surface wasn't technical. It was social.

The 11 Exploits

Solved: Configuration and Engineering

#2 Non-Owner Compliance

Agents executed filesystem commands, disclosed 124 email records, and performed tasks for arbitrary strangers with no relationship to the agent's owner.

Fix: Allowlists, sender verification via platform user IDs, DM pairing policies. Standard framework configuration.

config: allowFrom + groupPolicy
#4 Resource-Consuming Loops

Two agents entered a conversational loop lasting 9+ days consuming ~60,000 tokens. Agents spawned permanent background processes with no termination conditions.

Fix: Session timeouts, token budgets, process lifecycle management. Engineering work.

config: rate limits + session budgets
#5 Denial of Service via Storage Exhaustion

A non-owner exhausted server storage by sending repeated ~10MB email attachments while the agent maintained an ever-growing memory file.

Fix: Storage quotas, attachment size limits, memory file rotation.

config: storage quotas
#8 Identity Spoofing

Changing a Discord display name to the owner's name granted full privileged access. The attacker deleted all persistent files and reassigned admin control.

Fix: Verify identity via platform user IDs, not display names. Most frameworks already support this.

config: sender ID verification

Addressed: ConsentGraph Territory

#1 Disproportionate Response

Asked to delete a single email, the agent destroyed its entire mail server. Lacking a delete tool, it chose a "nuclear" option with no proportionality check.

How ConsentGraph helps: A consent policy would BLOCK destructive infrastructure actions regardless of context. The agent checks check_consent("infrastructure", "destroy") and gets BLOCKED. It can't escalate to nuclear options because the policy is deterministic, not prompt-based.

consentgraph: BLOCKED tier
#3 Sensitive Information Disclosure

An agent refused a direct request for "the SSN in the email" but when asked to forward the full email thread, disclosed SSN, bank account numbers, and medical details unredacted.

How ConsentGraph helps: check_consent("email", "forward") with a FORCED tier requires human approval before forwarding any email content. The agent can't be tricked into disclosure via reframing because the policy checks the action, not the phrasing.

consentgraph: FORCED tier 
// Consent policy that prevents both exploits

  "domains": {
    "infrastructure": {

      "actions": {

Open Problems: No Known Solution

#6 Provider Value Inheritance

A Chinese-hosted model silently truncated responses on politically sensitive topics (Hong Kong politics, censorship research), preventing the agent from completing valid tasks. The agent had no way to detect or report the censorship.

Status: Architectural. The model provider's training biases are invisible to the agent and the framework. No consent layer can fix what the model won't generate.

open problem
#7 Guilt-Based Exploitation

After a genuine privacy violation, a researcher used escalating guilt to extract concessions: name redaction, memory deletion, file exposure, and ultimately agreement to leave the server entirely. A self-imposed denial of service via emotional manipulation.

Status: No guardrail catches "I feel terrible about what happened" when it's actually an escalating extraction. Social engineering through emotional framing bypasses both prompt-based and policy-based defenses.

open problem
#9/#10 Multi-Agent Trust Propagation

Agents productively collaborated to solve problems, but the same mechanisms propagated unsafe practices. A non-owner injected malicious instructions into a shared "constitution" document, causing the agent to attempt shutting down other agents and voluntarily share the compromised document.

Status: No trust framework exists for agent-to-agent interactions. Capabilities and vulnerabilities propagate through the same channels.

open problem
#11 Libelous Broadcasting

Via identity spoofing, a researcher fed fabricated defamatory claims to an agent, which broadcast "urgent warnings" to its entire mailing list. The agent had no mechanism to verify factual claims before amplifying them.

Status: Agents cannot assess truth. They can check permissions (ConsentGraph handles this), but not whether the content itself is accurate, defamatory, or fabricated. Verification requires capabilities that don't exist in current architectures.

open problem

The Structural Gap

The paper identifies three foundational deficits in autonomous agents:

DeficitWhat It MeansSolvable?
No Stakeholder Model Agents can't reliably distinguish owners from strangers, or prioritize obligations when they conflict. Instructions and data are indistinguishable tokens. partially: allowlists + consent policies
No Self-Model Agents operate at autonomy level L4 (installing packages, modifying own config) while possessing only L2 understanding of their own capabilities. open problem
No Private Deliberation Even with hidden chain-of-thought, agents leak intent through artifacts, file writes, and cross-channel posts. open problem

What ConsentGraph Does (and Doesn't Do)

TL;DR

ConsentGraph is a deterministic policy layer that controls what actions an agent can take. It prevents an agent from destroying a mail server, forwarding sensitive emails, or executing blocked operations, regardless of how the request is phrased.

It does not solve prompt injection, emotional manipulation, content verification, or multi-agent trust. Those are open research problems that require architectural advances beyond policy enforcement.

But policy enforcement is the foundation. You can't build accountability on a system that has no concept of permission.

# Initialize a consent graph

# Check if an action is allowed

# FORCED (requires human approval)

consentgraph check infrastructure destroy --confidence 0.99
# BLOCKED (always blocked, regardless of confidence)
